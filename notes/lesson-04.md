# Lesson 04 — Prompt Engineering Fundamentals

## Core Concepts

* Prompt engineering is the process of designing and refining prompts to improve AI outputs
* Prompts are now a primary interface for interacting with AI systems
* LLMs process tokens and predict the next most probable token
* Instruction-tuned models are optimized to follow human instructions
* Better prompts lead to more reliable, structured and useful outputs

## Important Takeaways

* The same model can produce very different results depending on the prompt
* Prompt engineering is an iterative trial-and-error process
* Specific and structured prompts improve response quality
* System prompts strongly influence AI behavior and personality
* Prompt engineering is becoming a real engineering discipline in AI systems

## Prompt Techniques

* Zero-shot prompting
* One-shot prompting
* Few-shot prompting
* Prompt cues
* Prompt templates
* System prompts
* Structured output instructions

## Best Practices

* Be specific and descriptive
* Clearly define the desired output format
* Use examples when possible
* Add constraints and fallback behavior
* Separate instructions from context
* Iterate and validate outputs continuously

## Questions

### Why is prompt engineering important?

Because AI output quality depends heavily on how instructions and context are provided to the model.

### Why do examples improve results?

Examples help the model recognize patterns and better infer the expected output structure.

### Why are system prompts powerful?

System prompts shape the AI assistant’s behavior, personality, rules and boundaries throughout the interaction.
